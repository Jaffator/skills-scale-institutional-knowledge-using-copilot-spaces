# OctoAcme Project Management Docs

## Overview

OctoAcme uses a structured but lightweight project management approach to deliver high-quality features and services. Our process is built on five core principles:

- **Customer-first:** prioritize customer value and usability in every decision.
- **Iterative delivery:** ship small, testable increments and learn quickly.
- **Clear ownership:** every project has a named Project Manager (PM) and Product Manager (PdM) accountable for outcomes.
- **Data-informed decisions:** measure impact and iterate based on evidence, not assumption.
- **Psychological safety:** encourage open feedback, learning from failure, and continuous improvement.

The project lifecycle moves through six stages — **Initiation → Planning → Execution → Release → Risk Management → Retrospective** — each supported by actionable checklists, templates, and role-specific guidance documented in the process guides below.

## How to Use These Docs

**New team members:** Start with the [Project Management Overview](octoacme-project-management-overview.md) to understand OctoAcme's approach and guiding principles, then read the [Roles & Personas](octoacme-roles-and-personas.md) document to understand your role in the process.

**Active projects:** Use the stage-specific guides as checklists and reference material throughout the project lifecycle — from kicking off in [Initiation](octoacme-project-initiation.md) through closing out in [Retrospective & Continuous Improvement](octoacme-retrospective-and-continuous-improvement.md).

**Single source of truth:** This README is the front door to all OctoAcme project management process documentation. If you need to find a process guide, start here. If you use Copilot Spaces, add the relevant docs from this folder to your Space's context so Copilot can provide role- and process-specific guidance.
